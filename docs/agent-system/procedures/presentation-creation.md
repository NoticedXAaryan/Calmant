# Procedure: Presentation Creation

## Purpose

Create a complete presentation from the owner's request and deliver an artifact.

## Preferred Tool Order

1. Local PPTX generation.
2. Google Slides API.
3. Browser-based website automation only if requested or no API exists.

## Steps

1. Parse request:
   - topic,
   - audience,
   - slide count,
   - tone,
   - deadline,
   - required sources,
   - visual style.
2. If key inputs are missing and cannot be inferred, ask one concise clarification.
3. Research with cited sources.
4. Create outline.
5. Create slide-by-slide content.
6. Apply owner's presentation preferences from memory.
7. Generate deck artifact.
8. Render/export preview PDF or images.
9. QA:
   - every slide has title,
   - text fits,
   - citations included where required,
   - visuals are relevant,
   - no placeholder text.
10. Deliver artifact through Telegram/dashboard.

## Browser Website Variant

If using a presentation website:

1. Open site.
2. Login handoff if needed.
3. Create deck.
4. Screenshot previews.
5. Export/download.
6. Store artifact.

Final delivery must include the actual file or accessible link.

## Acceptance Criteria

- Deck is complete.
- No placeholders.
- Artifact exists.
- Owner receives final deck without needing to manually build it.
