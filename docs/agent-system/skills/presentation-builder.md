# Skill: Presentation Builder

## Purpose

Create complete presentation artifacts from a topic and requirements.

## When To Use

- User asks for slides, deck, pitch, report presentation, or lecture slides.

## Inputs

- topic,
- audience,
- slide count,
- style,
- source requirements,
- deadline,
- delivery format.

## Tools

- `web_search`
- `web_fetch`
- `presentation_create`
- `presentation_export_pdf`
- `artifact_create`
- optional `browser_*` tools for web-based builders.

## Procedure

1. Clarify only missing critical requirements.
2. Research topic if needed.
3. Create outline.
4. Draft slides.
5. Add visuals and citations.
6. Generate deck.
7. Export preview.
8. QA for completeness and layout.
9. Deliver artifact.

## Approval Gates

Approval required only for publishing/sharing externally.

## Outputs

- PPTX or Slides link,
- PDF preview,
- source list.

## Failure Handling

- Missing visual assets: use generated or open licensed assets with attribution.
- Export failure: deliver source deck and report issue.

## Tests

- no placeholder slides,
- exported file exists,
- text fits slides.
